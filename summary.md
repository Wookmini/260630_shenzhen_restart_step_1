# 심천지사 영수증 정산 자동화 시스템 작업 내역 요약 (Shenzhen Expense OCR System)

## 개요
심천지사 전도금 정산 시, 다량의 영수증(fapiao 등)의 판독, 대계정/소계정 분류, 금액 추출 및 엑셀 리포트 자동 생성을 수행하는 시스템의 버그 수정 및 안정화 작업을 진행했습니다.

## 주요 작업 내역
1. **USD 입금 / RMB 환전 데이터 정규화**
   - 입금 영수증을 처리할 때 기존에는 하나의 행에 묶이거나 금액이 중복 합산되는 문제가 있었습니다.
   - 이를 '입금 확인'과 '입금 수수료' 2개의 별도 행으로 분할 처리하도록 로직을 개선했습니다.
   - 웹앱 상단 합계(Total Amount)에 입금액이 합산되지 않도록 `amount`를 `None`으로 고정하여 회계 오류를 방지했습니다.

2. **교차검증 오류(파일명 불일치) 무한 캐시 버그 해결**
   - 파일 이름이 OCR 판독값과 맞지 않을 때 발생하는 `[파일명 교차검증 오류]`가 파일명이 정상적으로 수정된 후에도 계속 남는 버그를 해결했습니다.
   - `batch_processor.py`에서 기존 `data.json`의 캐시된 텍스트를 불러올 때 과거의 불일치 오류 텍스트는 필터링하여 지우고 새로운 검증 결과만 반영하도록 수정했습니다.
   - 추가로 지속적으로 파일 이름과 엑셀 데이터간의 매핑이 꼬이는 문제(정렬 순서에 따른 증빙 번호 할당 오류)를 해결하기 위해 엑셀 템플릿과 캐시 데이터를 완전히 지우고 원본 OCR 데이터 기준으로 파일명을 재정렬했습니다.

3. **웹앱 UI 경고창(Warning) 최적화**
   - 웹앱 목록 및 사이드바에서 정상적인 입금 확인 건(✅ 표시)이 오류처럼 붉은색 배경이나 아이콘으로 표시되던 로직을 제거했습니다.
   - 사용자 요청에 따라 불필요하게 표시되던 **"파표 외 영수증"** 및 **"파일명 교차검증 오류"** 메시지가 화면에 더 이상 출력되지 않도록 `app.js`와 `batch_processor.py` 양쪽에서 해당 텍스트 주입 로직을 삭제하거나 비활성화했습니다.

## 파일별 수정 사항 요약
- `_시스템_코어/batch_processor.py`: 입금 영수증 분할 처리, 교차검증 경고 무한 캐시 버그 수정, 불필요한 교차검증 경고 메시지 출력 주석 처리.
- `static/app.js`: 사이드바 렌더링 시 `✅` 포함된 성공 메시지는 에러에서 제외하도록 필터링 추가, `파표 외 영수증` 자동 주입 조건문 삭제.
- `작업장소 (영수증 보관)/2026-06/*`: 뒤죽박죽 섞여있던 테스트 영수증 파일들의 이름을 OCR 판독 데이터(대계정, 금액)와 완벽히 일치하도록 일괄 재정렬 및 정리.
