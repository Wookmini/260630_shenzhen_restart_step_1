## AI 분석 및 PRD 응답

### 배경 및 목표

현재 웹앱(SROT)은 OCR 업로드 중심으로 설계되어 있으나, 실제 운영 방식은 **파이썬 배치 실행 → 웹앱 검토/수정 → 엑셀 저장**의 흐름으로 정착되었습니다.  
이에 따라 불필요한 기능을 제거하고, **"검토 및 정산 확정" 중심의 고급 관리 대시보드**로 전면 리디자인합니다.

### 제거 대상 기능

| 기능 | 현재 위치 | 제거 사유 |
|------|----------|----------|
| 영수증 업로드 탭 | 탭 1 (📸) | 배치 처리로 대체 완료 |
| 썸네일 그리드 | 탭 1 내부 | 업로드 탭 제거로 함께 삭제 |
| OCR 원본 텍스트 필드 | 검토 폼 | 개발용 디버그 정보, 사용자 불필요 |
| 내보내기 탭 | 탭 4 (📥) | "엑셀 최종 저장" 버튼으로 통합 |
| 🧾 로고 아이콘 + "SROT" 텍스트 | 헤더 좌상단 | DB GlobalChip 로고로 교체 |
| `심천지사 영수증 OCR 변환 시스템` 서브타이틀 | 헤더 | 새 시스템명으로 교체 |

### V2 신규 설계

#### 헤더 구조

```
┌──────────────────────────────────────────────────────────────┐
│  [DB GlobalChip 로고]                                         │
│  심천지사 전도금 정산 자동화 시스템         [📅 월 선택 ▾]   │
│                                                              │
│  [📄 총 25건]  [✅ 처리 23건]  [⚠️ 확인필요 2건]  [💰 ¥13,676]│
│                                          [💾 엑셀 최종 저장] │
└──────────────────────────────────────────────────────────────┘
```

#### 탭 구조 (4탭 → 2탭으로 축소)

| 탭 번호 | 기존 | V2 |
|--------|------|-----|
| 1 | 📸 영수증 업로드 | ❌ 제거 |
| 2 | ✏️ 데이터 검토 | ✏️ 데이터 검토 (유지 + 개선) |
| 3 | 📊 정산 내역 | 📊 정산 내역 (기본 진입 탭으로 변경) |
| 4 | 📥 내보내기 | ❌ 제거 → 헤더 버튼으로 통합 |

#### 검토 폼 개선

**제거:** OCR 원본 텍스트 (textarea readonly)  
**추가:** 📝 비고 (textarea) → `data.json`의 `remark` 필드 ↔ 엑셀 S열 양방향 연동

**2컬럼 레이아웃:**
```
[유형]          [날짜]
[금액 (CNY)]    [담당자]
[내역]          [판매자]
[대계정]        [소계정]
[비고          (full-width)]
[💾 저장]  [🗑️ 삭제]
```

#### 담당자 자동 채우기 수정

- `/api/members` 기본 목록 + 현재 `receipts`에 등장하는 모든 `person` 값을 동적으로 드롭다운에 추가
- 영문명(`Lin Wei Jian`) 등 기존 멤버 목록에 없는 이름도 자동으로 select 옵션에 포함

#### 엑셀 저장 버튼

- 클릭 → `POST /api/months/{month}/save`
- 백엔드에서 `excel_exporter.py` 재실행 → 기존 파일 **제자리 덮어쓰기**
- 성공 토스트: `"✅ 엑셀 정산 파일 저장 완료 (YYYY-MM-DD HH:MM)"`

#### 디자인 시스템 (DB GlobalChip 브랜드 기반)

```css
--primary:     #1a7c3e;   /* DB GlobalChip 초록 */
--primary-lt:  #e8f5ee;   /* 연초록 배경 */
--accent:      #e05c1a;   /* DB GlobalChip 주황-레드 */
--accent-blue: #1a6cbf;   /* DB GlobalChip 블루 */
--surface:     rgba(255, 255, 255, 0.85);
--bg-base:     #f0f4f2;   /* 연한 그린 베이스 */
```

- Google Fonts: Pretendard (한글) + Inter (숫자/영문)
- 글래스모피즘: `backdrop-filter: blur(20px)`
- 마이크로 애니메이션: 탭 슬라이드, 뱃지 카운트업, 버튼 hover

## 교훈 및 인사이트

1. **운영 패턴 정착 후 UI 재설계**: 초기에는 업로드 중심 UI를 구축했으나, 실제 운영이 배치 처리로 정착되면서 UI의 절반이 불필요해졌음. **실제 사용 흐름을 파악한 후 리팩토링하는 것이 효과적**.

2. **브랜드 일치성**: 로고 이미지를 확인하여 디자인 팔레트를 브랜드 컬러로 통일함으로써 사내 시스템다운 고급스러움을 확보.

3. **기능 통합 vs 분리**: "내보내기 탭" → "헤더 버튼"으로 통합한 사례처럼, 핵심 액션은 항상 보이는 위치에 단일 CTA로 배치하는 것이 UX상 유리.

4. **비고 필드 양방향 연동**: 웹앱 비고 → 엑셀 S열, 엑셀 S열 → 웹앱 비고의 양방향 연동은 `remark` 필드를 `data.json`의 단일 소스로 관리하여 해결.
